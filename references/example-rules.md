# Worked Example — a Real "Standing Rules" Section

Below is an anonymized rules section from the student profile this skill was built around. Don't copy these rules to new users — they're one person's decisions. Use them to see the *level of specificity* the interview should reach: exact phrasing mandates, provenance notes, space allocations, confirm-every-time flags.

```markdown
## Resume-generation rules (standing)

1. **Grad date**: default Aug 2026; TBD — prompt me to confirm on EVERY resume.
2. **AI-assisted disclosure (CRITICAL)**: personal projects built via AI agent
   workflows. I own architecture, product direction, verification; agents write
   most implementation code. I hand-code C++/Python/SQL but not the full listed
   web/mobile stacks. Every resume with projects must keep the hands-on vs
   agent-directed skills split + a projects disclosure line. No first-person
   language in the agent-directed half.
3. **Never include spoken languages** — wasted space; proficiency lapsed.
4. **Whitespace**: if an item barely spills to a second line, condense to one;
   if it occupies ~1.5 lines, extend to fill 2 without filler. Coursework and
   skills lists always run to near end-of-line.
5. **Project space cap**: no project gets more resume space than my flagship
   project. Every project except one gets a GitHub link next to its header.
   Capitalization: always "GitHub", always "In Progress" for current courses.
6. **Excluded**: two discontinued projects (by name). Three specific roles
   appear on venture-framing resumes only.
7. Voice: bold, concise. Quantify only what's defensible. Never invent numbers.
8. **Mandatory on every resume**: my flagship summer program — always written
   out in full, NEVER the acronym, and ALWAYS its own standalone bullet with
   space allocated to signal importance (never merged into a combined
   programs line). Also mandatory: leadership program with its ~10% acceptance
   rate; service-prep program (max one line, can share a line); study abroad
   mentioned somewhere; my longest-tenured job (one line).
9. **Line-length calibration**: one-line bullets ≤96 chars; two-line bullets
   185–198. Verify with the ?audit loop before delivering.
10. **Fill the page**: I have ~2 resumes of content. If optimization shrinks
    the layout, ADD content until the page is vertically full. Target ≥95%.
11. Two framings: venture-led (work experience first) vs technical-led
    (projects first). Pick per role.
```

What makes these good:

- **Imperative and testable.** "Always written out in full, never the acronym" can be checked mechanically. "Make it look professional" cannot.
- **Provenance prevents regression.** Recording *why* something is banned or which old number is stale stops future sessions from reintroducing it.
- **Volatile facts are flagged, not resolved.** The grad date rule doesn't guess — it forces a confirmation every run.
- **Disclosure is pre-negotiated.** The AI-assisted framing was decided once, carefully, and now applies automatically — the user never has to re-justify it under deadline pressure.
- **Space is a first-class resource.** Caps, standalone-bullet mandates, and one-line limits encode the user's priorities into geometry.
